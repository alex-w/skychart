Version 4.2.1, Notes de la version

Nouvelles fonctions:
- Ajout d'une raquette virtuelle pour les télescopes ASCOM et INDI
- Modifie le bouton Voisinage pour obéir aux paramètres de la liste d'objets, augmente le rayon de recherche pour afficher au moins 10 objets
- Ajout d'objet à la liste d'observation à partir de la liste d'objets
- L'ancien pilote Encoder peut être installé depuis: https://github.com/pchev/alpaca_encoder/releases

Correction de bug:
- Les planètes en mode ligne doivent être vraiment transparentes
- Corrige "Voir les marques" pour la simulation des comètes et astéroïdes
- Corrige le lancement du navigateur Edge
- Affiche un message d'erreur clair lors d'une tentative d'exécution avec un affichage 8 ou 16 bits
- Corrige la déconnexion du télescope Alpaca


Version 4.2, Notes de la version

Nouvelles fonctions:

 - Nouvelle interface par défaut avec seulement deux barres de boutons.
 - Nouvelles icônes pour les boutons, mieux adaptée à la vision nocturne. Utilise une image plus grande pour les gros boutons et les écrans haute résolution.
 - Ajout d'une connexion directe au télescope Alpaca. Il est maintenant possible d'utiliser un télescope connecté à un ordinateur Windows ASCOM distant à partir de CdC fonctionnant sous Linux ou macOS.
 - Support du nouveau catalogue d'étoiles GAIA DR2.
 - Permet d'utiliser les catalogues d'étoiles HNSKY au format "290".
 - Nouvel outil de mosaïque pour préparer des mosaïques pour les applications de capture d’images.
 - Affiche le lever et le coucher des objets par rapport à l'horizon local si un fichier d'horizon est chargé. Ceci peut être désactivé dans Configuration / Observatoire. Le lever et coucher dans le calendrier sont toujours par rapport à l'horizon astronomique.
 - Définir les coordonnées de l'observatoire à partir du GPS du télescope, le nom de l'observatoire est "Télescope" et ne modifie pas les autres paramètres.
 - Nouveau bouton pour Park / Unpark du télescope ASCOM.
 - Permet un profil d'horizon d'observatoire par carte.
 - Ajout d'un menu de mise à jour pour Delta T et Jupiter GRS.
 - Ajout d'un mode de vision nocturne orange.
 - Réglage séparé des labels pour les astéroïdes et comètes.
 - Option permettant d'afficher uniquement la ligne de l'équateur céleste.
 - Option pour ne pas filtrer les nébuleuses brillantes par la magnitude, seulement par la taille.
 - Option pour définir la largeur de la ligne des dessin. Attention, cela ne fonctionne pas avec la ligne pointillée lors de l’impression sous Windows à cause des limitations de l’API.
 - Permet de verrouiller à la position définie dans le menu Voir / Position.

Correction et améliorations:

 - Le téléchargement des éléments des comète est désormais depuis http://astro.vanbuitenen.nl
 - La sélection du serveur et du pilote INDI est maintenant dans la boîte de connexion du télescope. L'option de lancement pour indistarter est supprimée.
 - Amélioration de l'affichage des étoiles doubles.
 - Amélioration du filtre de magnitude pour les petits champs.
 - La résolution de l'image d'arrière-plan est maintenant limitée pour améliorer les performances si l'écran est grand ou s'il n'y a pas assez de coeur cpu pour le traitement.
 - Amélioration de l'affichage de progression du téléchargement.
 - Affiche un avertissement dans le titre de la fenêtre si le programme est exécuté en tant qu’administrateur. CdC n’a jamais besoin de s’exécuter en tant qu’administrateur, mais cela peut casser la configuration.
 - Beaucoup de corrections de bugs.

Incompatibilité avec les versions précédentes:

 - Attention, la mise à jour des astéroïdes et des comètes depuis le menu principal supprime maintenant toutes les anciennes données de la base de données!
   Si vous devez conserver ces anciens éléments ou cumuler les données de plusieurs sources, vous devez utiliser la fonction du menu Configuration / Système solaire.
 - Supprime les pilotes directs LX200 et Encodeur qui sont obsolètes/non maintenables. Veuillez utiliser les pilotes ASCOM ou INDI correspondants.
   - https://ascom-standards.org/Downloads/ScopeDrivers.htm
   - https://www.indilib.org/devices/mounts.html
 - Déprécie l'utilisation de la base de données Mysql et suggère de passer à Sqlite si elle est actuellement utilisée. Le support de Mysql sera supprimé dans la prochaine version.
 - La communication client / serveur peut désormais utiliser les coordonnées J2000 au lieu du paramètre actuel de la carte. Vous pouvez modifier cette option dans Configuration / Général / Serveur.
 - Le fichier de cercle enregistré (.cdcc) inclut l'équinoxe des coordonnées sur la première ligne et davantage d'informations pour l'utilisation avec l'outil Mosaïque.
